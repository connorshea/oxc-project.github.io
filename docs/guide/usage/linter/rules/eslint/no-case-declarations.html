eslint/no-case-declarations Pedantic â€‹

What it does â€‹

Disallow lexical declarations in case clauses.

Why is this bad? â€‹

The reason is that the lexical declaration is visible in the entire switch block but it only gets initialized when it is assigned, which will only happen if the case where it is defined is reached.

Examples â€‹

Examples of incorrect code for this rule:

javascript
switch (foo) {
  case 1:
    let x = 1;
    break;
  case 2:
    const y = 2;
    break;
  case 3:
    function f() {}
    break;
  default:
    class C {}
}

Examples of correct code for this rule:

javascript
switch (foo) {
  case 1: {
    let x = 1;
    break;
  }
  case 2: {
    const y = 2;
    break;
  }
  case 3: {
    function f() {}
    break;
  }
  default: {
    class C {}
  }
}

How to use â€‹

To enable this rule in the CLI or using the config file, you can use:

bash
oxlint --deny no-case-declarations
json
{
  "rules": {
    "no-case-declarations": "error"
  }
}

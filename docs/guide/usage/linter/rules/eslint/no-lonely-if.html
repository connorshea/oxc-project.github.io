eslint/no-lonely-if Pedantic â€‹

ðŸš§ An auto-fix is still under development.

What it does â€‹

Disallow if statements as the only statement in else blocks

Why is this bad? â€‹

When an if statement is the only statement in an else block, it is often clearer to use an else if instead.

Examples â€‹

Examples of incorrect code for this rule:

js
if (condition) {
  // ...
} else {
  if (anotherCondition) {
    // ...
  }
}
js
if (condition) {
  // ...
} else {
  if (anotherCondition) {
    // ...
  } else {
    // ...
  }
}

Examples of correct code for this rule:

js
if (condition) {
  // ...
} else if (anotherCondition) {
  // ...
}
js
if (condition) {
  // ...
} else if (anotherCondition) {
  // ...
} else {
  // ...
}
js
if (condition) {
  // ...
} else {
  if (anotherCondition) {
    // ...
  }
  doSomething();
}

How to use â€‹

To enable this rule in the CLI or using the config file, you can use:

bash
oxlint --deny no-lonely-if
json
{
  "rules": {
    "no-lonely-if": "error"
  }
}
